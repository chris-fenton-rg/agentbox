title: Run an agent
description: Launch Claude Code, Codex, OpenCode, or pi in a box and attach or detach sessions

AgentBox runs **Claude Code**, **OpenAI Codex**, **OpenCode**, and **pi** (Earendil pi coding agent) inside a box, in a detachable tmux session — so you can leave the agent working, detach, and reattach later.

The four agent verbs (`claude`, `codex`, `opencode`, `pi`) each create the box if it doesn't already exist, start the agent, and attach you. They share all `agentbox create` flags — see [CLI commands](/docs/cli) — plus session-resume and attach controls. This page assumes you've read [core concepts](/docs/core-concepts); the project-teleport mechanics live in [teleport a project](/docs/teleport-a-project).

<Callout title="NOTE">
`<box>` is optional almost everywhere — it defaults to the single box for the current project. Otherwise pass its short index (`1`, `2`, …), name, or id prefix.
</Callout>

## Launch an agent

`agentbox claude` is the one-liner: it creates a box (snapshotting the current directory), starts Claude Code in a detached tmux session, and attaches you. `agentbox codex`, `agentbox opencode`, and `agentbox pi` have the identical shape.

```bash
agentbox claude                       # create a box + start Claude Code, then attach
agentbox claude -- --model sonnet     # pass extra args through to claude
agentbox pi                           # create a box + start pi, then attach
agentbox pi -- --model zai-glm/glm-5.2  # pass extra args through to pi
```

Target another directory with `-w`, and put pass-through args after `--`. Boxes get an auto-numbered name by default; pass `-n <name>` to label one. The first run pulls or builds the `agentbox/box:dev` image (~1 GB, one-time) and offers a setup wizard that installs project deps and your dev server. See [local Docker](/docs/local-docker) for the image and [services and tasks](/docs/services-and-tasks) for the wizard.

```console
$ agentbox claude
log: ~/.agentbox/logs/claude.log
Starting Claude in a box...
> Run setup wizard? › Yes
  ...box ready...
attaching — Control+a d to detach, leaves claude running
```

## Detach and reattach

The agent runs in a detachable tmux session whose prefix is **`Control+a`** (not the usual `Ctrl-b`). Press `Control+a` then `d` to detach — the agent keeps running in the box.

`Control+a` also opens an actions menu: `c` (VS Code / Cursor), `s` (noVNC screen), `u` (box URL), **`t` (a fresh shell in the *same* box, opened in a new tab)**, and **`k` (destroy the box, after a confirmation)**. The shell tab opens as a cmux surface in the current workspace, a new tmux window, or a new iTerm2 tab depending on your terminal.

Reattach with `agentbox attach` — it picks the running agent session in the box automatically. Resync your host config and (re)start a session with `agentbox claude start` (or `codex start` / `opencode start` / `pi start`).

```bash
# detach from inside the session: press Control+a then d
agentbox attach 1                     # reattach to whichever agent is running on box 1
agentbox claude start mybox           # start a session (resyncs host ~/.claude)
agentbox pi start mybox               # start a pi session (resyncs host ~/.pi/agent)
```

`agentbox attach` is agent-agnostic — if multiple agent sessions live in the same box it prompts you to pick. It never auto-starts: when nothing is running it prints a warning and exits non-zero. To start a fresh session use the agent's own command (`agentbox claude`, `codex`, `opencode`, `pi`) — `start` also resyncs your host config so new skills, plugins, or a fresh login flow into the box. For `--attach-in`, `--no-attach`, and the other attach controls, see [CLI commands](/docs/cli).

<Callout type="warn" title="HEADS UP">
`attach` never resyncs your host `~/.claude` (skills, plugins, login). After installing a new skill or signing in on the host, use `claude start` (or a fresh `agentbox claude`) so the box picks it up.
</Callout>

## Switch agents

Codex, OpenCode, and pi follow the identical flow — `agentbox codex` / `agentbox opencode` / `agentbox pi`, each with `attach` and `start` subcommands. (`codex` and `opencode` also have a `login` subcommand; pi does not — see [First-run auth](#first-run-auth) below.)

```bash
agentbox codex                        # OpenAI Codex in a box
agentbox opencode                     # OpenCode in a box
agentbox pi                           # pi (Earendil pi coding agent) in a box
agentbox dashboard                    # switch between running boxes/agents
```

Auto-approve is on by default because the box is isolated from your host; pass `--no-dangerously-skip-permissions` to keep the agent's own approval prompts. Each agent shares an identity volume across boxes by default — opt out per box with `--isolate-claude-config` (and the codex/opencode/pi equivalents; `--isolate-pi-config` opts into a per-box `agentbox-pi-config-<id>` volume). See [configuration](/docs/configuration) and [CLI commands](/docs/cli).

You can run different agents in different boxes for the same project and switch between them with `agentbox dashboard`. Claude and Codex resume sessions; OpenCode does not yet (resume starts a fresh box).

<Callout type="warn" title="HEADS UP">
`agentbox pi` runs on **Docker and Daytona** (`--provider daytona`, requires `agentbox prepare --provider daytona`). Hetzner, Vercel, and E2B are not yet supported. Session teleport (`-c`/`--resume` carrying a host session into a fresh box) is also not yet supported for pi and emits a friendly error; pi's own `--continue` flag still works inside the box.
</Callout>

<Callout title="WHY">
Auto-approve is safe to default on because the box can't touch your machine — it's isolated from the host. The agent works without stopping to ask, and you keep your laptop clean.
</Callout>

## First-run auth

Credentials are handled on the host and seeded into every box (macOS Keychain doesn't transfer into containers). On the first `agentbox claude` with no credentials, AgentBox offers an interactive sign-in that's saved and reused by every future box. Skip it with `-y` (or in CI) and claude will prompt you to `/login` inside the box.

```bash
agentbox claude login                 # sign in once, reused by every box
agentbox codex login                  # device-auth flow for Codex
agentbox opencode login -- --provider anthropic
```

```console
$ agentbox claude
? Sign in with your Claude subscription? (saved and reused by every box) › Yes
✔ Signed in with your Claude subscription — saved for future boxes.
```

Setting `ANTHROPIC_API_KEY` / `CLAUDE_CODE_OAUTH_TOKEN` (Claude) or `OPENAI_API_KEY` (Codex) on the host forwards it into the box and skips the sign-in offer. Host `~/.claude` stays the source of truth — every `create` / `claude` syncs it in, so skills, plugins, MCP, and login flow into the next box. Cloud providers capture the login to `~/.agentbox` and push it in on create — see [hetzner](/docs/hetzner), [daytona](/docs/daytona), and [vercel](/docs/vercel).

### pi auth

pi has no interactive login command (`agentbox pi login` does not exist). Authenticate pi one of two ways:

1. **API-key env vars** — set any of `ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, `OPENROUTER_API_KEY`, `GEMINI_API_KEY`, `GOOGLE_API_KEY`, `GROQ_API_KEY`, `ZAI_API_KEY`, `ZAI_GLM_API_KEY`, `MOONSHOT_API_KEY`, or `KIMI_API_KEY` on the host and AgentBox forwards them into every box automatically.

2. **OAuth tokens in `~/.pi/agent/auth.json`** — AgentBox syncs your host's `~/.pi/agent/` directory into the box on every `pi` or `pi start` run. Sign in on your host with pi's own auth flow, then create or start a box and the token travels with it.

pi config lives under `~/.pi/agent/` (`auth.json`, `settings.json`, `models.json`, `extensions/`). AgentBox mounts a shared Docker volume `agentbox-pi-config` at `/home/vscode/.pi` and syncs the host's `~/.pi/agent` into its `agent/` subdir; `PI_CODING_AGENT_DIR` pins pi's config directory inside the box. Use `--isolate-pi-config` (or config key `box.isolatePiConfig`) to opt into a per-box volume instead of the shared one.

```bash
# pull box-side pi config back to host ~/.pi/agent (additive, never overwrites)
agentbox download pi
```

<Callout title="NOTE">
pi is installed into the box at runtime via `npm install -g --ignore-scripts @earendil-works/pi-coding-agent`. The base image ships Node 24 so no extra setup is needed; pi is not baked into the published base image.
</Callout>

<Callout title="TIP">
Sign in once on the host. Install a skill or sign in there, and the next box picks it up automatically. See [configuration](/docs/configuration) and [sync and git](/docs/sync-and-git).
</Callout>
